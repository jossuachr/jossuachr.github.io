DataFest 2025: SMS Spam Classification

Proud winners of Best in Show at DataFest Mizzou 2025 with our SMS spam detection pipeline, built end-to-end in R using tidymodels.

Project Overview

The goal was to accurately classify SMS messages as spam or ham. We leveraged the publicly available SMS Spam Collection dataset (5,574 messages) and implemented a reproducible workflow:

  1. Data import and cleaning
  2. Exploratory data analysis (EDA)
  3. Text preprocessing (tokenization, stop-word removal, TF-IDF)
  4. Model tuning with random forest (ranger engine)
  5. Final evaluation and interpretation

Text Preprocessing

Using textrecipes, we built a recipe that:

  • Tokenizes text into individual words (`step_tokenize`)
  • Removes English stop words (`step_stopwords`)
  • Filters to the top 1,000 most frequent tokens (`step_tokenfilter`)
  • Transforms to TF-IDF features (`step_tfidf`)

Modeling & Tuning

We specified a Random Forest classifier with 1,000 trees and tuned two hyperparameters (mtry and min_n) using 5-fold cross-validation:

The best combination (mtry = 20, min_n = 4) yielded a mean ROC AUC of 0.986 on validation folds.

Conclusions

This pipeline achieved >97% accuracy and highlighted key terms like free, call, and claim as strong spam indicators. The modular recipe and workflow ensure reproducibility and easy extension (e.g., adding alternative classifiers or deploying via Shiny).
