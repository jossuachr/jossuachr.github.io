Winner of Best in Show

May 2025 • RStudio Wizards at Truman State University

Our team tackled an industry-provided dataset to uncover key drivers of rental pricing, build predictive models, and deliver an interactive dashboard—earning us the Best in Show award at DataFest Mizzou 2025.

1. Data & Tools

We used R and the following packages:

  • tidyverse for data wrangling and visualization
  • lubridate for date handling
  • ranger and randomForest for modeling
  • iml for interpretability (VIP & ALE)
  • sf & leaflet for mapping
  • shiny for the interactive app

# Mode helper
get_mode <- function(x) {
  ux <- unique(x)
  ux[which.max(tabulate(match(x, ux)))]
}

# Summarize by lease_id
leases_clean <- leases_full %n%  
  group_by(lease_id) %n%  
  summarize(overall_rent = sum(rent_amount), .groups='drop') %n%  
  left_join(leases_full, by="lease_id") %n%  
  mutate(
    price_category = ifelse(predicted_rent > overall_rent, "Under", "Over")
  )

4. Modeling

We built a Random Forest regression model, tuning mtry and min.node.size over a random grid via 5-fold cross-validation.

Show Modeling Code 
# Split data
set.seed(123)
split_data <- initial_split(leases_clean, strata=price_category)
train_data <- training(split_data)
test_data <- testing(split_data)

# Recipe
t_rec <- recipe(overall_rent ~ unemployment_rate + avg_occupancy_proportion + leasedSF + RBA + availability_proportion, data=train_data) %n%  
  step_impute_median(all_numeric_predictors()) %n%  
  step_normalize(all_numeric_predictors())

# Model spec & workflow
rf_spec <- rand_forest(mode="regression", trees=1000, mtry=tune(), min_n=tune()) %>% set_engine("ranger")
rf_wf <- workflow() %>% add_recipe(t_rec) %>% add_model(rf_spec)

# Tuning
grid <- grid_random(mtry(range=c(2,5)), min_n(range=c(5,20)), size=20)
cv <- vfold_cv(train_data, v=5)
tune_res <- tune_grid(rf_wf, resamples=cv, grid=grid, metrics=metric_set(rmse, rsq))

best <- select_best(tune_res, "rmse")
final_wf <- finalize_workflow(rf_wf, best)
final_fit <- last_fit(final_wf, split_data)
